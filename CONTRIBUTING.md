# Contributing to Emotional Messaging Helper

Thank you for your interest in contributing to Emotional Messaging Helper!

## Development Setup

1. Clone the repository
2. Open the project in Android Studio
3. Make sure you have Ollama running locally with a model (llama3.2 or better)
4. Run the app on a physical device (emulators have limited accessibility support)

## Code Style

- Follow Kotlin coding conventions
- Use meaningful variable and function names
- Prefer composition over inheritance
- Keep functions small and focused

## Pull Request Process

1. Create a new branch from `main` for your feature/fix
2. Make your changes
3. Update documentation if needed
4. Open a Pull Request with a clear description
5. Link any related issues

## Reporting Issues

Please use the issue templates when reporting bugs or requesting features.
